# 分层架构指南

## 架构总览

源系统 → irpt(导入层) → dwi(明细整合层) → dwm(中间汇总层) → dws(汇总宽表层) → ads(应用层) → 报表/BI

## irpt 导入层

**前缀**: irpt_
**说明**: 从源系统直接导入，不进行业务清洗
**表类型**:
- irpt_{粒度}_{业务域}_amt — 事实数据
- irpt_{粒度}_{业务域}_amt_his — 历史归档
- irpt_{实体}_his — 维度主数据

## dwi 明细整合层

**前缀**: dwi_
**说明**: 经清洗、去重、标准化、补全后的明细数据
**特点**:
- 窄表细粒度
- 保留所有维度外键
- CASE WHEN 做编码翻译
- LEFT JOIN 补齐维度描述

## dwm 中间汇总层

**前缀**: dwm_
**说明**: 按中间粒度轻度聚合
**特点**:
- 减少后续计算量
- 按部门/产品/区域等维度预聚合

## dws 汇总宽表层

**前缀**: dws_
**说明**: 面向主题的宽表，一个业务主题一张表
**特点**:
- 宽表大宽列
- 含累计值（累计同期、累计实际、年度累计等）
- 常用窗口函数（SUM over partition by order by）

## ads 应用层

**前缀**: ads_
**说明**: 直接供报表、BI使用的数据集
